Programme Python pour Élèves

Partie 2 : Approfondissement et Projets (Séances 31 à 60)

Module 7 : Fonctions et Modularité

31

Mes premières fonctions - def

Description : On arrête de se répéter ! Une "fonction", c'est comme créer notre propre sortilège de code. On lui donne un nom, on y met des instructions, et ensuite, on peut l'appeler autant de fois qu'on veut, rendant notre code plus court et plus propre.
Objectif : Savoir regrouper un bloc de code dans une fonction avec def et l'appeler pour l'exécuter.
Plan de la séance (1h30) :
  • Le principe DRY (Don't Repeat Yourself) : Montrer un code répétitif et l'améliorer (15 min).
  • Pratique guidée : Créer une fonction saluer() et l'appeler plusieurs fois (25 min).
  • Défi : Transformer un ancien exercice en fonction dessiner_maison() (30 min).
  • Discussion : Les avantages des fonctions (organisation, lisibilité) (20 min).
32

Paramètres et arguments

Description : On peut rendre nos fonctions plus intelligentes en leur donnant des "paramètres". C'est comme donner des instructions spécifiques à notre sortilège. Au lieu de dire toujours "Bonjour", on peut dire "Bonjour" à un nom spécifique.
Objectif : Créer des fonctions flexibles qui acceptent des données (arguments) pour personnaliser leur comportement.
Plan de la séance (1h30) :
  • Analogie de la machine à soda : le paramètre est le bouton qu'on choisit (15 min).
  • Pratique guidée : Créer une fonction saluer_personne(nom) (30 min).
  • Défi : Créer une fonction addition(a, b) qui prend deux nombres et affiche leur somme (45 min).
33

Valeurs de retour - return

Description : Parfois, on veut qu'une fonction nous "rende" un résultat qu'on peut stocker. C'est le rôle du mot-clé magique return. La fonction devient une usine qui produit une valeur.
Objectif : Savoir créer des fonctions qui produisent un résultat utilisable par le reste du programme.
Plan de la séance (1h30) :
  • Différence entre print (montrer) et return (donner) (20 min).
  • Pratique : Transformer addition(a, b) pour qu'elle retourne le résultat (30 min).
  • Activité : Utiliser le résultat d'une fonction dans une variable (resultat = addition(5, 3)) (20 min).
  • Défi : Créer une fonction calculer_prix_ttc(prix_ht) qui retourne le prix avec TVA (20 min).
34

Fonctions et listes

Description : Les fonctions sont parfaites pour travailler avec des listes. On peut créer des outils sur-mesure pour trouver la plus grande valeur, calculer une moyenne, ou filtrer des éléments.
Objectif : Savoir passer une liste en paramètre d'une fonction et la manipuler.
Plan de la séance (1h30) :
  • Exemple : Créer une fonction afficher_liste(ma_liste) (25 min).
  • Défi 1 : Créer une fonction calculer_moyenne(liste_notes) qui retourne la moyenne (30 min).
  • Défi 2 : Créer une fonction trouver_plus_grand(nombres) qui retourne le max d'une liste (35 min).
35

Portée des variables - local vs global

Description : Une variable créée dans une fonction est "locale" : elle est piégée à l'intérieur. Une variable créée à l'extérieur est "globale". Comprendre cette différence est essentiel pour éviter des bugs bizarres.
Objectif : Comprendre la "visibilité" d'une variable et la différence entre portée locale et globale.
Plan de la séance (1h30) :
  • Analogie de la "bulle" de la fonction : ce qui est dedans reste dedans (20 min).
  • Démonstration : Tenter d'accéder à une variable locale depuis l'extérieur et voir l'erreur (25 min).
  • Discussion : Pourquoi cette "protection" est une bonne chose ? (20 min).
  • Exercice de débogage : Trouver pourquoi une variable n'est pas accessible (25 min).
Note : C'est un concept abstrait. Le débogueur visuel de Thonny est très utile pour voir les variables apparaître et disparaître.
36

Projet - Calculatrice avancée modulaire

Description : On reconstruit notre calculatrice de manière professionnelle. Chaque opération sera sa propre fonction. Le programme principal appellera la bonne fonction selon le choix de l'utilisateur.
Objectif : Organiser un programme complexe en le découpant en petites fonctions logiques.
Plan de la séance (1h30) :
  • Conception : Lister toutes les fonctions nécessaires (addition, soustraction, afficher_menu) (20 min).
  • Développement : Écrire chaque fonction, puis les assembler dans la boucle principale (50 min).
  • Bonus : Ajouter une fonction pour la puissance (**) (20 min).

Module 8 : Dictionnaires et Structures de Données

37

Les dictionnaires - Clés et valeurs

Description : Pour ranger des informations par étiquette (ex: "nom" -> "Alex"), on utilise un "dictionnaire". C'est un carnet magique où chaque information (la valeur) est rangée derrière une étiquette unique (la clé).
Objectif : Créer un dictionnaire pour stocker des données structurées et savoir retrouver une valeur en utilisant sa clé.
Plan de la séance (1h30) :
  • Analogie du dictionnaire (mot/clé et définition/valeur) (20 min).
  • Pratique : Créer un dictionnaire pour un personnage {'nom': 'Pikachu', 'type': 'Electrique', 'pv': 50} (30 min).
  • Accès : Récupérer une valeur (perso['type']) (20 min).
  • Défi : Créer un mini-traducteur avec un dictionnaire (20 min).
38

Manipuler les dictionnaires

Description : On peut ajouter, modifier, ou supprimer des paires clé-valeur. On apprendra aussi à parcourir les dictionnaires avec des boucles pour voir tout leur contenu.
Objectif : Maîtriser les opérations de base et savoir itérer sur les clés ou les valeurs.
Plan de la séance (1h30) :
  • Opérations : Ajouter une entrée, modifier une valeur, supprimer avec del (30 min).
  • Boucles : Parcourir avec for cle in mon_dico: et for cle, valeur in mon_dico.items(): (30 min).
  • Défi : Créer un gestionnaire de scores pour un jeu (clés=noms, valeurs=scores) (30 min).
39

Dictionnaires imbriqués - Structures complexes

Description : On peut mettre des dictionnaires dans des dictionnaires ! C'est comme ça qu'on représente des données complexes, comme un profil utilisateur avec une section "adresse" qui est elle-même un dictionnaire.
Objectif : Savoir créer et naviguer dans des structures de données hiérarchiques.
Plan de la séance (1h30) :
  • Exemple : Créer un dictionnaire pour une classe, où chaque élève est un dictionnaire (30 min).
  • Navigation : Accéder à une info nichée (ex: classe['eleve1']['notes']['maths']) (30 min).
  • Défi : Créer une base de données de super-héros, chacun avec un dictionnaire de pouvoirs (30 min).
40

Listes et dictionnaires ensemble

Description : La combinaison ultime ! Une liste de dictionnaires (ex: une liste d'élèves) ou un dictionnaire contenant des listes (ex: un élève avec une liste de matières). C'est le secret pour organiser presque n'importe quelle donnée.
Objectif : Savoir combiner les deux structures pour modéliser des situations complexes.
Plan de la séance (1h30) :
  • Exemple 1 : Une liste de dictionnaires (catalogue de produits) (25 min).
  • Exemple 2 : Un dictionnaire contenant des listes (recette de cuisine) (25 min).
  • Défi : Créer une bibliothèque (liste de livres, chaque livre est un dictionnaire) (40 min).
41

Algorithmes sur dictionnaires

Description : Maintenant qu'on sait structurer des données, on peut écrire des algorithmes pour les analyser : trouver l'élève avec la meilleure note, calculer le prix total d'un panier, etc.
Objectif : Écrire du code qui parcourt des structures complexes pour en extraire des informations.
Plan de la séance (1h30) :
  • Problème guidé : Calculer le montant total d'un panier (liste de dictionnaires de produits) (40 min).
  • Défi : À partir d'un dictionnaire de scores, trouver et afficher le nom du joueur avec le meilleur score (50 min).
42

Projet - Base de données interactive

Description : On met tout en œuvre pour créer une application de gestion, en utilisant des dictionnaires pour stocker des informations riches sur chaque élément.
Objectif : Créer un programme de gestion (CRUD: Create, Read, Update, Delete) en utilisant une liste de dictionnaires.
Plan de la séance (1h30) :
  • Conception : Adapter le projet de la séance 30 pour utiliser des dictionnaires (20 min).
  • Développement du "Gestionnaire de bibliothèque personnelle" (60 min).
  • Discussion : Comment sauvegarder ces données ? (Transition) (10 min).

Module 9 : Fichiers et Persistance

43

Lire des fichiers texte

Description : Comment faire pour que notre programme lise un texte depuis un fichier ? On apprend à ouvrir un fichier, à lire son contenu ligne par ligne ou en entier, pour l'utiliser dans notre code.
Objectif : Savoir ouvrir un fichier en mode lecture et en extraire les données textuelles.
Plan de la séance (1h30) :
  • Concept : Le principe de la persistance des données (10 min).
  • Pratique : Utiliser with open('fichier.txt', 'r') as f: pour lire et afficher un fichier (35 min).
  • Défi : Créer un "analyseur de texte" qui compte le nombre de lignes ou de mots dans un fichier (45 min).
44

Écrire dans des fichiers

Description : On apprend à sauvegarder des informations (un score, une note, un journal) dans un fichier pour qu'elles ne disparaissent pas quand on ferme le programme. C'est le secret des sauvegardes de jeux !
Objectif : Savoir ouvrir un fichier en mode écriture ('w') ou ajout ('a') pour y sauvegarder du texte.
Plan de la séance (1h30) :
  • Concept : Les modes 'w' (write/écraser) et 'a' (append/ajouter) (20 min).
  • Pratique : Créer un fichier "journal.txt" et y écrire une phrase (25 min).
  • Défi : Créer un carnet de notes qui demande une note à l'utilisateur et l'ajoute à la fin d'un fichier "notes.txt" (45 min).
45

Fichiers CSV - Données structurées

Description : Le format CSV (Comma-Separated Values) est un standard pour stocker des données tabulaires, comme dans un tableur. C'est juste du texte avec des virgules, très simple à lire et à écrire pour nos programmes.
Objectif : Savoir lire et écrire des données structurées dans un fichier CSV.
Plan de la séance (1h30) :
  • Explication du format CSV (15 min).
  • Pratique : Lire un fichier CSV (ex: nom,email,tel) et utiliser line.split(',') pour le transformer en liste (35 min).
  • Défi : Créer un gestionnaire de contacts qui sauvegarde les données (nom, tel) dans un fichier CSV (40 min).
46

Gestion d'erreurs avec les fichiers

Description : Que se passe-t-il si on essaie de lire un fichier qui n'existe pas ? Crash ! On apprend à utiliser try...except pour "essayer" une opération risquée et "attraper" l'erreur si elle se produit, sans faire planter le programme.
Objectif : Rendre les programmes robustes en gérant l'erreur FileNotFoundError.
Plan de la séance (1h30) :
  • Démonstration d'un crash en tentant d'ouvrir un fichier inexistant (15 min).
  • Pratique : Encadrer l'ouverture du fichier avec un bloc try...except (35 min).
  • Défi : Améliorer notre gestionnaire de contacts pour qu'il crée le fichier s'il n'existe pas, au lieu de planter (40 min).
47

Projets avec persistance

Description : On applique nos nouvelles compétences de sauvegarde à nos anciens projets. Un système de "high scores" qui se souvient du meilleur score entre les parties est un excellent exemple.

  • Discussion : Quelles autres applications pourraient bénéficier de la sauvegarde ? (To-do list, inventaire...) (30 min).

Module 10 : Programmation Orientée Objet (POO)

49

Introduction aux classes - Mes premiers objets

Description : On passe au niveau supérieur ! Une "classe" est le plan de construction, le moule, pour fabriquer un type d'objet (ex: un Monstre). Chaque monstre créé (l'objet) aura ses propres caractéristiques, basées sur le même plan.
Objectif : Comprendre la différence entre une classe (le plan) et un objet (la chose construite), et créer sa première classe.
Plan de la séance (1h30) :
  • Analogie du moule à gâteaux (la classe) et des gâteaux (les objets) (20 min).
  • Pratique : Créer une classe Robot vide. Puis créer deux "instances" (objets) : r1 = Robot() et r2 = Robot() (35 min).
  • Défi : Créer une classe Voiture et fabriquer trois voitures différentes (35 min).
50

Attributs et méthodes

Description : On ajoute des détails à notre plan. Les "attributs" sont les données de l'objet (ex: la couleur d'une voiture). Les "méthodes" sont ses actions (ex: une voiture peut démarrer ou accélérer).
Objectif : Définir les propriétés (attributs) et les actions (méthodes) d'une classe.
Plan de la séance (1h30) :
  • Pratique : Ajouter des attributs (nom, pv) et une méthode attaquer() à une classe Monstre (40 min).
  • Concept clé : Le mot-clé self qui représente "l'objet lui-même" (20 min).
  • Défi : Créer une classe CompteBancaire avec un attribut solde et des méthodes deposer() et retirer() (30 min).
51

Constructeur et initialisation - __init__

Description : Le constructeur __init__ est une méthode spéciale qui est appelée automatiquement quand on crée un nouvel objet. C'est là qu'on définit ses attributs de départ.
Objectif : Utiliser __init__ pour initialiser des objets avec des valeurs personnalisées dès leur création.
Plan de la séance (1h30) :
  • Explication du rôle du constructeur (15 min).
  • Pratique : Modifier la classe Voiture pour qu'on puisse créer une voiture en spécifiant sa marque et sa couleur dès le début (ma_voiture = Voiture("Tesla", "Rouge")) (35 min).
  • Défi : Créer une classe Joueur avec un nom et des points de vie définis à la création (40 min).
52

Méthodes spéciales et représentation - __str__

Description : Quand on fait print(mon_objet), Python affiche quelque chose de moche. Avec la méthode spéciale __str__, on peut définir exactement comment notre objet doit s'afficher, pour un rendu beaucoup plus propre.
Objectif : Utiliser __str__ pour fournir une représentation textuelle claire et lisible d'un objet.
Plan de la séance (1h30) :
  • Démonstration de l'affichage par défaut d'un objet (20 min).
  • Pratique : Ajouter une méthode __str__ à notre classe Joueur pour qu'elle affiche "Joueur: Alex (100 PV)" (35 min).
  • Défi : Améliorer toutes les classes créées précédemment avec une méthode __str__ utile (35 min).
53

Listes d'objets

Description : C'est ici que la POO devient surpuissante. On peut créer des listes d'objets ! Une liste d'élèves, une équipe de joueurs, une armée de monstres... Cela permet de gérer des collections d'entités complexes.
Objectif : Savoir créer et manipuler une liste contenant des instances de nos propres classes.
Plan de la séance (1h30) :
  • Pratique guidée : Créer une liste de plusieurs objets Joueur, puis utiliser une boucle pour afficher les informations de chaque joueur (40 min).
  • Défi : Créer une classe Ecole qui contient une liste d'objets Etudiant. Ajouter une méthode pour inscrire un nouvel étudiant (l'ajouter à la liste) (50 min).
54

Projet - Application orientée objet

Description : Projet final de POO : un mini jeu de rôle textuel. On créera une classe Joueur et une classe Monstre, et on les fera interagir dans un système de combat simple.
Objectif : Organiser un programme complexe en utilisant plusieurs classes qui interagissent entre elles.
Plan de la séance (1h30) :
  • Conception : Définir les attributs et méthodes pour les classes Joueur et Monstre (20 min).
  • Développement guidé du système de combat (un joueur et un monstre s'attaquent tour à tour dans une boucle while) (60 min).
  • Bonus : Ajouter une classe Objet (ex: Potion) que le joueur peut utiliser (10 min).

Module 11 : Projets Créatifs et Graphiques

55

Introduction à turtle - Dessiner avec Python

Description : Marre du texte ? On va enfin faire du dessin ! Le module turtle nous donne une "tortue" virtuelle qui tient un crayon. On peut lui donner des ordres simples ("avance", "tourne") pour créer des dessins incroyables.
Objectif : Savoir importer turtle, créer une fenêtre de dessin et commander la tortue pour dessiner une forme simple.
Plan de la séance (1h30) :
  • Démonstration d'un dessin complexe fait avec turtle pour motiver (10 min).
  • Pratique : Faire avancer la tortue, la faire tourner. Dessiner un carré pas à pas (35 min).
  • Exploration : Découvrir color(), pensize(), penup(), pendown() (25 min).
  • Défi : Dessiner une maison (carré + triangle) avec des couleurs différentes (20 min).
56

Animations et interactions avec turtle

Description : Un dessin statique, c'est bien. Un dessin qui bouge, c'est mieux ! On va apprendre à faire réagir notre tortue aux touches du clavier et à créer des animations simples.
Objectif : Créer des animations simples et contrôler un objet turtle avec le clavier.
Plan de la séance (1h30) :
  • Concept : La boucle de jeu (while True) pour l'animation (20 min).
  • Pratique : Faire bouger une tortue en continu (25 min).
  • Interaction : Utiliser onkey() pour lier des fonctions aux touches du clavier (25 min).
  • Défi : La célèbre "Course de tortues" où chaque joueur contrôle une tortue avec des touches différentes (20 min).
57

Projet créatif - Turtle Art

Description : On laisse libre cours à notre créativité ! En combinant les boucles, les fonctions et le module turtle, on peut créer des œuvres d'art complexes et magnifiques, comme des rosaces ou des fractales simples.
Objectif : Utiliser la programmation pour générer des motifs artistiques complexes.
Plan de la séance (1h30) :
  • Exemple : Montrer comment créer une rosace en répétant le dessin d'un carré et en tournant légèrement à chaque fois (30 min).
  • Projet libre : Laisser les élèves expérimenter pour créer leur propre "générateur d'art". Ils peuvent jouer avec les couleurs, les formes, les angles (60 min).
58

Introduction à une bibliothèque de jeu (Pygame Zero)

Description : turtle est super pour dessiner, mais pour les vrais jeux, il faut un outil plus puissant. Pygame Zero est une bibliothèque conçue pour les débutants qui simplifie énormément la création de jeux avec des images (sprites), des sons et des contrôles fluides.
Objectif : Comprendre la structure de base d'un jeu Pygame Zero (draw(), update()) et savoir afficher un personnage (Actor) à l'écran.
Plan de la séance (1h30) :
  • Installation de Pygame Zero (pip install pgzero) (20 min).
  • Structure de base : Créer un fichier avec les fonctions draw() et update() (25 min).
  • Pratique : Créer un Actor avec une image et le dessiner à l'écran. Le faire bouger avec les flèches du clavier dans la fonction update() (45 min).
59

Projet - Jeu simple (Pygame Zero)

Description : On construit notre premier vrai jeu graphique ! Un personnage doit se déplacer pour attraper des objets qui apparaissent au hasard. On gère le mouvement, la détection des collisions et le score.
Objectif : Créer un jeu complet et jouable avec Pygame Zero.
Plan de la séance (1h30) :
  • Conception du jeu "Attrape la pomme" (15 min).
  • Développement guidé : Mouvement du joueur, apparition aléatoire de la pomme (35 min).
  • Collision : Utiliser colliderect() pour détecter quand le joueur touche la pomme, augmenter le score et déplacer la pomme (40 min).

Projet Final et Présentation

Description : La grande finale ! Chaque élève a le choix : soit améliorer son jeu existant (ajouter un chronomètre, des ennemis, des sons), soit commencer un nouveau mini-projet. Le plus important est de présenter sa création aux autres, d'expliquer une partie de son code et d'être fier du chemin parcouru.
Objectif : Consolider ses connaissances en réalisant un projet personnel et s'exercer à présenter son travail technique.
Plan de la séance (1h30) :
  • Phase de projet libre : Amélioration ou création (50 min).
  • Tour de table : Chaque élève présente son écran, montre son jeu et explique la fonctionnalité dont il est le plus fier (30 min).
  • Conclusion et ouverture : Que peut-on faire ensuite avec Python ? (Science des données, web, etc.) (10 min).
Note : C'est une séance de célébration. L'objectif n'est pas d'évaluer sévèrement mais de valoriser le travail, la créativité et la résolution de problèmes dont les élèves ont fait preuve tout au long du programme.
