Programme Python pour Élèves

Partie 1 : Fondations et Bases (Séances 1 à 30)

Module 1 : Découverte et Installation

1

Qu'est-ce que la programmation ?

Description : On découvre le monde secret du code. Programmer, c'est comme apprendre une nouvelle langue pour parler aux ordinateurs et leur donner des ordres très précis, un peu comme une recette de cuisine magique.
Objectif : Comprendre le rôle d'un programmeur et réaliser que le code est partout autour de nous (smartphones, jeux vidéo, etc.).
Plan de la séance (1h30) :
  • Jeu de rôle "L'élève-robot" : Suivre des instructions ultra-précises pour dessiner une forme (25 min).
  • Discussion : Où trouve-t-on du code dans la vie de tous les jours ? (15 min).
  • Découverte : Présentation de Python comme un langage "facile à lire" (15 min).
  • Activité papier : Écrire l'algorithme pour une tâche simple (ex: faire un sandwich) (35 min).
2

Installation et premier contact avec Python

Description : On met en place notre atelier de programmation ! On installe Python et un outil très simple appelé "Thonny". Ensuite, on écrit notre toute première ligne de code pour faire dire "Hello, World!" à l'ordinateur. C'est un moment magique !
Objectif : Avoir un environnement de développement fonctionnel et exécuter avec succès son premier programme.
Plan de la séance (1h30) :
  • Installation guidée de Python et Thonny sur les postes (40 min).
  • Exploration de l'interface de Thonny : la zone de code, la console (15 min).
  • Le grand moment : Écrire et exécuter print("Hello, World!") (15 min).
  • Défi : Personnaliser le message avec son nom ou une blague (20 min).
3

L'environnement de développement

Description : Un bon artisan connaît ses outils. On apprend à bien utiliser notre éditeur, à sauvegarder notre travail dans des dossiers bien rangés, et surtout, à ne pas paniquer quand on voit un message d'erreur rouge !
Objectif : Devenir à l'aise avec l'éditeur, savoir organiser ses fichiers et commencer à déchiffrer les messages d'erreur simples.
Plan de la séance (1h30) :
  • Organisation : Créer un dossier "Projets_Python" sur son ordinateur (10 min).
  • Pratique : Écrire un programme, le sauvegarder, le fermer et le rouvrir (20 min).
  • Le jeu de la "Chasse aux bugs" : Donner des petits bouts de code avec des erreurs simples (ex: parenthèse manquante) à corriger (40 min).
  • Discussion : Un message d'erreur n'est pas un échec, c'est un indice ! (20 min).

Module 2 : Variables et Types de données

4

Les variables - Mes premières boîtes à données

Description : Un programme a besoin de se souvenir d'informations. Pour cela, on utilise des "variables", qui sont comme des boîtes avec des étiquettes. On peut y ranger un nom, un score, ou un mot de passe.
Objectif : Savoir créer une variable pour stocker une information et l'afficher.
Plan de la séance (1h30) :
  • Analogie : Les boîtes magiques étiquetées (15 min).
  • Pratique guidée : Créer des variables nom = "Alex", age = 12 et les afficher (30 min).
  • Règles : Comment bien nommer ses variables (ex: couleur_preferee) (15 min).
  • Défi : Créer un profil personnel avec des variables pour le nom, l'âge et le jeu vidéo préféré (30 min).
5

Les nombres en Python

Description : Python est un génie des maths ! On va découvrir comment il manipule les nombres entiers (int) et les nombres à virgule (float), et comment on peut lui faire faire toutes les opérations de base (+, -, *, /).
Objectif : Utiliser Python comme une calculatrice puissante pour résoudre des problèmes mathématiques simples.
Plan de la séance (1h30) :
  • Exploration : Utiliser la console Python pour des calculs instantanés (20 min).
  • Pratique : Écrire un script qui calcule et affiche le résultat d'une opération (25 min).
  • Concept : La différence entre 10 (entier) et 10.0 (décimal) (15 min).
  • Défi : Créer un programme qui calcule son âge en jours, puis en heures (30 min).
6

Les chaînes de caractères (strings)

Description : Les programmes ne manipulent pas que des nombres, ils manipulent aussi du texte ! On appelle ça des "chaînes de caractères" ou strings. On va apprendre à les coller ensemble pour former des phrases.
Objectif : Savoir créer des variables contenant du texte et les combiner pour créer des messages personnalisés.
Plan de la séance (1h30) :
  • Concept : Les guillemets qui délimitent le texte (10 min).
  • Concaténation : Utiliser le signe + pour coller des chaînes de caractères (30 min).
  • Jeu : "Le constructeur de phrases folles" où on combine des variables de texte (30 min).
  • Défi : Créer un générateur de cartes de vœux qui combine un nom et un message (20 min).
7

Les booléens - Vrai ou Faux

Description : Comment un ordinateur prend-il une décision ? Il se pose des questions dont la réponse est soit VRAI (True), soit FAUX (False). Ces deux valeurs sont la base de toute la logique en programmation.
Objectif : Comprendre les valeurs True et False et savoir utiliser les opérateurs de comparaison (==, !=, >, <).
Plan de la séance (1h30) :
  • Jeu : "Vrai ou Faux ?" avec des affirmations simples (ex: "Le ciel est vert") (15 min).
  • Pratique : Taper des comparaisons dans la console Python (5 > 3, 10 == 10) et voir le résultat (30 min).
  • Concept clé : La différence entre = (assigner) et == (comparer) (15 min).
  • Défi : Écrire un programme qui compare deux âges et affiche True si le premier est plus grand (30 min).
8

Révision et projet - Mon premier programme complet

Description : C'est l'heure de rassembler toutes nos connaissances ! On va construire notre premier vrai petit projet qui utilise des variables, des nombres, du texte et des booléens pour créer une "carte d'identité" interactive.
Objectif : Synthétiser les notions des 4 types de données dans un seul programme cohérent.
Plan de la séance (1h30) :
  • Révision rapide des 4 types de données (15 min).
  • Présentation du projet "Carte d'identité interactive" (10 min).
  • Développement guidé du projet (45 min).
  • Présentation des projets entre élèves et suggestions d'amélioration (20 min).
Note : C'est la première fois qu'ils combinent tout. L'entraide et le "pair programming" (programmer à deux) sont très encouragés ici.

Module 3 : Entrées/Sorties et Interaction

9

Faire parler mon programme - print() avancé

Description : On sait déjà afficher du texte, mais on peut le faire avec style ! On va découvrir comment afficher plusieurs informations sur une même ligne et comment utiliser les "f-strings" pour insérer facilement nos variables dans du texte.
Objectif : Maîtriser le formatage de texte pour créer des affichages clairs et professionnels, comme des menus ou des rapports.
Plan de la séance (1h30) :
  • Rappel de print() et de la concaténation (15 min).
  • Introduction aux f-strings : print(f"Bonjour {nom}!"). C'est magique ! (30 min).
  • Activité : Créer de l'art ASCII simple (dessins avec des caractères) en utilisant plusieurs print() (25 min).
  • Défi : Utiliser des f-strings pour créer et afficher un menu de restaurant propre et bien aligné (20 min).
10

Écouter l'utilisateur - input()

Description : Un programme devient vraiment intéressant quand il peut interagir avec nous. Avec la commande input(), notre programme va pouvoir nous poser des questions et utiliser nos réponses.
Objectif : Savoir créer un programme qui dialogue avec l'utilisateur, en récupérant du texte et en le convertissant en nombre si besoin.
Plan de la séance (1h30) :
  • Démonstration d'un programme qui pose une question (10 min).
  • Pratique : Utiliser input() pour récupérer un nom et afficher un message de bienvenue personnalisé (25 min).
  • Concept clé : input() renvoie toujours du texte ! Apprendre à le convertir en nombre avec int() (25 min).
  • Défi : Créer une mini-calculatrice qui demande deux nombres à l'utilisateur et affiche leur somme (30 min).
11

Validation et gestion des erreurs simples

Description : Que se passe-t-il si l'utilisateur tape du texte alors que notre programme attend un nombre ? Crash ! On va apprendre à anticiper ces erreurs et à rendre nos programmes plus solides en vérifiant ce que l'utilisateur nous donne.
Objectif : Savoir vérifier si l'entrée d'un utilisateur est du type attendu pour éviter que le programme ne plante.
Plan de la séance (1h30) :
  • Démonstration : Faire planter volontairement la calculatrice de la séance précédente (10 min).
  • Introduction à la méthode .isdigit() pour vérifier si une chaîne de caractères est un nombre (30 min).
  • Pratique : Sécuriser la calculatrice en utilisant une condition if pour vérifier l'entrée avant de la convertir (35 min).
  • Défi : Améliorer le programme pour qu'il affiche un message d'erreur clair si l'utilisateur se trompe (15 min).
12

Projet interactif - Questionnaire personnalisé

Description : On met en pratique tout ce qu'on a appris sur l'interaction ! On va créer un questionnaire complet qui pose plusieurs questions à l'utilisateur, stocke ses réponses dans des variables et affiche un résumé personnalisé à la fin.
Objectif : Construire un programme interactif de A à Z, en gérant les entrées, les variables et les affichages de manière cohérente.
Plan de la séance (1h30) :
  • Brainstorming et conception du questionnaire sur papier (20 min).
  • Développement guidé du programme, question par question (45 min).
  • Phase de test : les élèves échangent leurs programmes et les testent (15 min).
  • Amélioration basée sur les retours (10 min).

Module 4 : Conditions et Logique

13

Les conditions - if, mon premier choix

Description : On donne un cerveau à notre programme ! Avec if, il peut prendre une décision. "SI la réponse est bonne, ALORS ajouter un point". C'est le début des programmes vraiment intelligents.
Objectif : Écrire un programme qui exécute un bloc de code seulement si une condition est remplie.
Plan de la séance (1h30) :
  • Jeu de rôle "Si... Alors..." (ex: "Si je dis 'Python', levez la main") (15 min).
  • Pratique : Écrire un vérificateur de mot de passe simple (30 min).
  • Concept clé : L'indentation (le décalage) est cruciale après un if (15 min).
  • Défi : Créer un programme qui autorise l'accès à un jeu vidéo seulement si le joueur a plus de 12 ans (30 min).
14

if-else - Deux chemins possibles

Description : Parfois, on veut faire une action si la condition est vraie, et une AUTRE action si elle est fausse. C'est le rôle de else. "SI le mot de passe est bon, ALORS 'Bienvenue', SINON 'Accès refusé'".
Objectif : Savoir créer des programmes qui ont deux branches logiques différentes.
Plan de la séance (1h30) :
  • Explication du concept avec un organigramme simple (15 min).
  • Pratique : Améliorer le vérificateur de mot de passe avec un message d'erreur grâce à else (25 min).
  • Défi 1 : Créer un programme qui dit si un nombre est pair ou impair (25 min).
  • Défi 2 : Créer un jeu de pile ou face simple (25 min).
15

elif - Plusieurs choix possibles

Description : Et si on a plus de deux choix ? "SI la note est > 15, 'Excellent', SINON SI > 10, 'Bien', SINON 'À revoir'". C'est là que elif (contraction de "else if") entre en jeu pour créer de véritables menus ou des logiques complexes.
Objectif : Gérer des situations avec de multiples conditions en chaîne.
Plan de la séance (1h30) :
  • Analogie du "triage de colis" selon la destination (15 min).
  • Pratique : Créer un programme qui donne une mention en fonction d'une note (Excellent, Bien, Passable, Insuffisant) (35 min).
  • Défi : Créer un menu pour une calculatrice qui demande à l'utilisateur de choisir entre addition, soustraction, etc. (40 min).
16

Conditions complexes - and, or, not

Description : On peut rendre nos conditions encore plus puissantes en les combinant. Avec and (ET), les deux conditions doivent être vraies. Avec or (OU), une seule suffit. Avec not (NON), on inverse le résultat.
Objectif : Savoir construire des conditions sophistiquées en combinant plusieurs tests logiques.
Plan de la séance (1h30) :
  • Explication avec des phrases simples : "Pour sortir, il faut faire ses devoirs ET ranger sa chambre" (20 min).
  • Pratique avec and : Créer un accès sécurisé qui vérifie l'âge ET un mot de passe (25 min).
  • Pratique avec or : Créer un programme qui donne une réduction si on est étudiant OU si c'est le week-end (25 min).
  • Défi : Combiner les trois pour un système de connexion encore plus complexe (20 min).
17

Conditions imbriquées

Description : On peut mettre des conditions à l'intérieur d'autres conditions, comme des poupées russes ! Cela permet de créer des logiques très fines, idéales pour les jeux d'aventure textuels.
Objectif : Structurer une logique complexe en plaçant des blocs if/else à l'intérieur d'autres blocs.
Plan de la séance (1h30) :
  • Schéma au tableau d'un arbre de décision (15 min).
  • Pratique guidée : "SI tu vas dans la forêt, ALORS (SI tu rencontres un ours, ALORS...)" (35 min).
  • Défi : Créer le début d'un mini-jeu d'aventure textuel avec au moins deux niveaux de choix imbriqués (40 min).
Note : L'indentation devient ici absolument cruciale et peut être une source de confusion. Il faut y aller pas à pas et vérifier le code souvent.
18

Projet - Jeu de questions-réponses

Description : Grand projet de logique ! On va créer un quiz complet. Le programme posera des questions, vérifiera les réponses avec if/elif/else, et tiendra le score.
Objectif : Intégrer toutes les notions de conditions vues dans le module pour créer un jeu fonctionnel.
Plan de la séance (1h30) :
  • Conception : Choisir un thème pour le quiz et écrire 5 questions/réponses (20 min).
  • Développement : Coder le quiz, en gérant le score avec une variable qui s'incrémente (50 min).
  • Bonus : Ajouter des niveaux de difficulté qui changent les questions (utilisant une condition au début) (20 min).

Module 5 : Boucles et Répétition

19

La boucle while - Répéter tant que...

Description : La boucle while est comme un chien de garde tenace. TANT QU'UNE condition est vraie, elle répète un bloc de code. Parfait pour un jeu qui continue TANT QUE le joueur a des vies.
Objectif : Savoir créer des répétitions qui dépendent d'une condition et faire attention à ne pas créer de boucle infinie.
Plan de la séance (1h30) :
  • Concept : "Tant que tu n'as pas fini tes devoirs, tu restes à ton bureau" (15 min).
  • Pratique : Créer un compte à rebours de 10 à 1 avec une boucle while (25 min).
  • Le piège ! Créer volontairement une boucle infinie et apprendre à l'arrêter (Ctrl+C) (15 min).
  • Défi : Créer un jeu qui demande un mot de passe et qui continue de le demander TANT QUE la réponse est fausse (35 min).
20

La boucle for - Répéter un nombre de fois

Description : Si on sait exactement combien de fois on veut répéter une action (ex: 10 fois), la boucle for est notre meilleure amie. C'est l'outil parfait pour les tâches répétitives et prévisibles.
Objectif : Savoir utiliser for ... in range(...) pour exécuter un bloc de code un nombre précis de fois.
Plan de la séance (1h30) :
  • Comparaison : Quand utiliser while vs for ? (15 min).
  • Pratique guidée : Afficher les nombres de 0 à 9 avec for i in range(10): (25 min).
  • Exploration : Les options de range() (commencer à un autre nombre, avancer de 2 en 2) (20 min).
  • Défi : Créer un programme qui demande un nombre à l'utilisateur et affiche sa table de multiplication (30 min).
21

Boucles imbriquées - Boucles dans des boucles

Description : On peut mettre une boucle à l'intérieur d'une autre boucle ! C'est comme ça qu'on peut parcourir une grille (chaque ligne, puis chaque colonne) ou créer des motifs complexes en art ASCII.
Objectif : Comprendre comment fonctionne une boucle imbriquée pour générer des structures en deux dimensions.
Plan de la séance (1h30) :
  • Analogie de l'horloge : La trotteuse (boucle interne) fait un tour complet pour chaque avancée de la grande aiguille (boucle externe) (20 min).
  • Pratique : Afficher les coordonnées d'une grille 3x3 ((0,0), (0,1)...) (30 min).
  • Défi : Utiliser des boucles imbriquées et print() pour dessiner un carré ou un triangle rempli d'étoiles `*` (40 min).
22

break et continue - Contrôler les boucles

Description : Parfois, on veut sortir d'une boucle avant la fin (break) ou sauter une étape et passer à la suivante (continue). Ce sont nos jokers pour gérer des cas spéciaux.
Objectif : Savoir utiliser break pour arrêter une boucle et continue pour sauter une itération.
Plan de la séance (1h30) :
  • Concept de break : "Cherche dans cette pile de livres jusqu'à ce que tu trouves le bon, puis ARRÊTE" (20 min).
  • Pratique avec break : Améliorer le jeu de devinettes pour qu'il s'arrête dès que le joueur trouve la bonne réponse (25 min).
  • Concept de continue : "Affiche tous les nombres de 1 à 10, SAUF le 5" (20 min).
  • Pratique avec continue : Écrire une boucle qui affiche tous les nombres impairs entre 1 et 20 (25 min).
23

Boucles et conditions ensemble

Description : C'est ici que la vraie puissance se révèle. En mettant des conditions if à l'intérieur de nos boucles, on peut créer des programmes qui analysent des données, filtrent des informations et résolvent des problèmes complexes.
Objectif : Combiner boucles et conditions pour résoudre des problèmes logiques.
Plan de la séance (1h30) :
  • Exemple guidé : Parcourir les nombres de 1 à 100 et afficher "Fizz" pour les multiples de 3, "Buzz" pour les multiples de 5 (le fameux FizzBuzz) (40 min).
  • Défi : Créer un programme qui demande 5 notes à l'utilisateur (avec une boucle) et qui compte combien de notes sont au-dessus de la moyenne (10/20) (50 min).
24

Projet - Jeu complet avec boucles

Description : Projet final pour ce module : le célèbre jeu du Pendu ! Ce jeu est parfait car il utilise tout : une boucle while pour le jeu principal, des conditions pour vérifier les lettres, et la gestion du texte.
Objectif : Créer une version textuelle complète du jeu du Pendu.
Plan de la séance (1h30) :
  • Décomposition du jeu en petites étapes (choisir un mot, boucle de jeu, demander une lettre, vérifier, afficher...) (20 min).
  • Développement guidé du cœur du jeu (50 min).
  • Amélioration : Gérer un nombre de vies qui diminue et afficher le dessin du pendu en art ASCII (20 min).

Module 6 : Listes et Collections

25

Les listes - Mes premières collections

Description : Comment stocker une collection d'objets, comme tous les ingrédients d'une recette ou tous les monstres d'un niveau ? On utilise une "liste" ! C'est comme un sac à dos magique où l'on peut ranger plein d'informations en ordre.
Objectif : Créer une liste, y ajouter des éléments et accéder à un élément précis grâce à son index.
Plan de la séance (1h30) :
  • Analogie : La liste de courses ou l'inventaire d'un jeu (15 min).
  • Pratique : Créer une liste de ses 3 meilleurs amis. Afficher la liste complète, puis seulement le premier ami (index [0]) (30 min).
  • Concept clé : Le comptage à partir de ZÉRO ! (20 min).
  • Défi : Créer une playlist musicale sous forme de liste et afficher "La chanson en cours est : ..." (25 min).
26

Manipuler les listes - append, remove, pop

Description : Une liste n'est pas statique, elle vit ! On va apprendre les sorts pour ajouter un élément à la fin (append), en enlever un par son nom (remove) ou par sa position (pop).
Objectif : Savoir modifier dynamiquement une liste en ajoutant ou en retirant des éléments.
Plan de la séance (1h30) :
  • Démonstration des 3 méthodes principales (20 min).
  • Pratique guidée : Créer un inventaire de jeu, y ajouter une "épée" avec append, puis retirer une "branche" avec remove (30 min).
  • Défi : Créer une "To-do list" interactive où l'utilisateur peut ajouter une tâche, ou marquer une tâche comme faite (la retirer de la liste) (40 min).
27

Parcourir les listes avec des boucles

Description : Comment afficher chaque élément d'une liste sans connaître sa taille ? On utilise une boucle for ! C'est la manière la plus élégante et la plus puissante de traiter tous les éléments d'une collection, un par un.
Objectif : Maîtriser la boucle for element in ma_liste: pour effectuer une opération sur chaque item d'une liste.
Plan de la séance (1h30) :
  • Explication : Montrer la syntaxe simple et élégante (15 min).
  • Pratique : Créer une liste de courses et l'afficher joliment, un article par ligne, avec une boucle (25 min).
  • Défi 1 : Créer une liste de notes et utiliser une boucle pour calculer la somme totale, puis la moyenne (25 min).
  • Défi 2 : Améliorer le jeu du Pendu en utilisant une liste pour les lettres déjà devinées par le joueur (25 min).
28

Listes de listes - Structures 2D

Description : Comment représenter un plateau de jeu, comme un échiquier ou une grille de morpion ? En mettant des listes à l'intérieur d'une autre liste ! On crée ainsi une structure en 2D, avec des lignes et des colonnes.
Objectif : Comprendre comment créer et accéder aux éléments d'une grille 2D.
Plan de la séance (1h30) :
  • Visualisation au tableau d'une grille et de ses coordonnées (ligne, colonne) (20 min).
  • Pratique : Créer une grille de morpion 3x3 et apprendre à accéder à une case précise (ex: grille[1][2]) (30 min).
  • Défi : Écrire une fonction qui prend une grille en paramètre et l'affiche joliment dans la console en utilisant des boucles imbriquées (40 min).
29

Algorithmes sur listes - tri, recherche

Description : Python nous offre des outils magiques pour travailler sur les listes. On va découvrir comment trier une liste d'un coup (.sort()), et comment vérifier si un élément est présent (avec l'opérateur in).
Objectif : Savoir utiliser les fonctions de tri intégrées et l'opérateur de recherche in pour manipuler efficacement des collections.
Plan de la séance (1h30) :
  • Démonstration de .sort() sur une liste de nombres, puis de noms (20 min).
  • Pratique avec in : Améliorer le jeu du Pendu pour vérifier si la lettre proposée a déjà été jouée (if lettre in lettres_jouees:) (30 min).
  • Défi : Créer un programme qui demande à l'utilisateur d'entrer 5 noms, les stocke dans une liste, puis affiche la liste triée par ordre alphabétique (40 min).
30

Projet final Partie 1 - Application de gestion

Description : C'est le grand final de la première partie ! On va construire une application complète qui utilise tout ce qu'on a vu : variables, conditions, boucles et surtout des listes pour gérer une collection (de livres, de films, de jeux...).
Objectif : Créer un programme piloté par un menu qui permet à l'utilisateur d'ajouter, de voir, et de supprimer des éléments d'une collection.
Plan de la séance (1h30) :
  • Conception du "Gestionnaire de collection" : définir les fonctionnalités (menu principal, ajouter, lister, supprimer, quitter) (20 min).
  • Développement guidé du programme, fonctionnalité par fonctionnalité (60 min).
  • Présentation des projets et discussion sur les améliorations possibles pour la Partie 2 (ex: sauvegarder dans un fichier) (10 min).
Note : Ce projet est une excellente évaluation des compétences acquises. Il est complexe mais très gratifiant, car il ressemble à une "vraie" application.
